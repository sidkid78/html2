Introduction

In the world of professional maintenance, there is a fundamental gap between general mechanical knowledge and site-specific technical expertise. A master technician may understand the principles of internal combustion, but without the specific torque specifications for a 2018 Camry’s cylinder head or the proprietary error codes of a niche industrial generator, they are often left to educated guesswork.

This 'mechanic’s dilemma' is the catalyst for Listen & Fix, a high-performance multimodal diagnostic platform. By leveraging the Google Gemini 3 Flash model, Listen & Fix serves as an automated acoustic and visual diagnostic engine, integrating audio, video, and image analysis with a sophisticated 'just-in-time' Retrieval-Augmented Generation (RAG) pipeline.

This system doesn't just process data; it perceives it. It hears the subtle frequency of a failing bearing and sees the thermal signature of a circuit overload, then instantly cross-references these symptoms against a live-crawled library of technical manuals. The result is a transition from diagnostic uncertainty to precision repair, providing users with an authoritative 'Digital Foreman' that bridges the gap between general AI capability and granular mechanical mastery.

The Senses of the Machine: Multimodal Fusion

The core of the Listen & Fix ecosystem lies in its ability to bypass traditional manual feature extraction. Instead of relying on isolated spectral analysis or static image recognition, the system utilizes Gemini’s native multimodal capabilities to correlate disparate data streams in real-time.

The diagnostic engine captures raw acoustic signatures (44.1 kHz / 24-bit) to identify specific mechanical pathologies such as harmonic resonance, valve tapping, or belt squeals. Simultaneously, it synchronizes visual frames from high-resolution video to detect leaks, wear patterns, or fraying components.

This multimodal fusion allows the AI to identify micro-fractures or bearing wear before catastrophic failure occurs by correlating high-frequency audio spikes with visual thermal data. Every diagnosis is delivered with a confidence score (0.0 to 1.0) and a severity rating, ensuring that the user understands the urgency and reliability of the AI’s findings.

The Just-in-Time RAG Infrastructure

To ensure that every repair guide is grounded in factory-accurate data, Listen & Fix employs a dual-path RAG service. This infrastructure is managed by two primary entities: the 'Scout' (DocumentCrawlerService) and the 'Librarian' (RAGService).

The Scout scours trusted sources like iFixit, RepairPal, and the NHTSA to ingest specific service manuals tailored to the user's equipment. The Librarian then 'tears the manual down to the block,' using the gemini-embedding-2-preview model to convert technical text into high-dimensional vectors.

This allows the AI to link a user's description of an 'engine knock' to a manual's specific 'connecting rod clearance' specs, even if the terminology differs. By indexing data with smart chunking logic that respects sentence and paragraph boundaries, the system ensures that critical safety warnings and torque settings are never lost in the shuffle.

The Digital Foreman Design System

Trust is the primary currency in high-stakes mechanical repair. To facilitate this, the Listen & Fix UI follows the 'Digital Foreman' design system—an aesthetic rooted in authority, clarity, and organization. The interface avoids the visual clutter of traditional apps, utilizing a 'No-Line Rule' where space is defined through tonal shifts rather than borders.

A dual-typeface system using Inter and Public Sans ensures maximum legibility in workshop environments. The user journey is structured through a four-step React-based wizard:

  • Capture: High-resolution media upload.
  • Details: A 6-step context wizard to define make, model, and skill level.
  • Analyzing: Real-time progress updates via an NDJSON stream.
  • Results: An interactive RepairGuide dashboard featuring localized parts sourcing and step-by-step instructions.

This workflow transforms a chaotic troubleshooting session into a calibrated, linear process.

Technical Architecture and Implementation

The ecosystem is built on a Next.js framework, utilizing the @google/genai SDK for seamless integration. The backend is orchestrated through a series of TypeScript services that handle everything from multimodal processing to background document ingestion. To ensure reliable data parsing, the system utilizes Gemini's JSON mode.

import { GoogleGenerativeAI } from "@google/generative-ai";

export async function analyzeMechanicalIssue(audioData: string, imageData: string[], equipmentMeta: any) {
  const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY);
  
  // Using Gemini 3 Flash with JSON response schema
  const model = genAI.getGenerativeModel({
    model: "gemini-3-flash-preview",
    generationConfig: { responseMimeType: "application/json" }
  });

  const prompt = `Analyze this ${equipmentMeta.make} ${equipmentMeta.model}.
    Identify the source of the sound in the audio and correlate with visual wear in images.
    Return a JSON object with the following keys: "diagnosis", "confidence_score", "severity_level", and "suggested_fix".`;

  const result = await model.generateContent([
    prompt,
    { inlineData: { data: audioData, mimeType: "audio/wav" } },
    ...imageData.map(img => ({ inlineData: { data: img, mimeType: "image/jpeg" } }))
  ]);

  return JSON.parse(result.response.text());
}

This architecture ensures that large file uploads (up to 2GB) are handled efficiently via resumable upload protocols, while the DocumentCrawlerService works in the background to fetch specific technical specs.

Safety Protocols and Domain Awareness

Safety is not an afterthought in the Listen & Fix ecosystem; it is a dynamic component of the diagnostic output. The system identifies high-risk equipment and injects mandatory safety protocols tailored to the specific domain.

For vehicle diagnostics, the system generates alerts regarding jack stands and hazardous fluids. For electrical or HVAC repairs, it mandates Lock-Out/Tag-Out (LOTO) procedures and highlights EPA certification requirements for refrigerant handling.

These warnings are rendered in a sophisticated 'burnt orange' (#5f2200) to ensure they carry the appropriate editorial weight without causing user panic. By adjusting the complexity of instructions based on the user's self-reported skill level (Beginner to Professional), the system ensures that the guidance is both actionable and safe for the specific person performing the repair.
