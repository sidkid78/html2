Domain Report

Protein Biological Process Annotation Pipeline

Executive Summary

This report synthesizes the development of a self-contained machine learning pipeline designed to predict Biological Process (BP) Gene Ontology (GO) terms from amino acid sequences. The pipeline integrates state-of-the-art protein language models with a specialized evaluation framework designed to optimize the maximum information-accretion weighted F1-measure (wFmax). Key components include ESM-2 sequence embeddings, a hierarchical Information Content (IC) weighting system, a custom multi-label neural network architecture, and a knowledge-tiered evaluation module.

1. Information Content and Semantic Weighting

The foundation of the pipeline lies in its ability to handle the hierarchical and non-uniform nature of Gene Ontology terms. Worker-Biol-1 implemented a frequency-based calculation for Information Content (IC), defined as:

IC = -log₁₀(p(t))

where p(t) represents the probability of a term t occurring within the dataset.

Crucially, the system implements hierarchical propagation, ensuring that the frequency of a specific term includes the counts of all its descendant terms in the GO graph. This methodology ensures that rare, highly specific biological processes are assigned higher weights than generic, high-level terms (e.g., "cellular process"). These weights are serialized into a centralized data structure used for both the loss function and final evaluation metrics.

2. Sequence Feature Engineering via ESM-2

To bridge the gap between raw amino acid sequences and numerical machine learning inputs, the pipeline utilizes Meta AI’s ESM-2 (Evolutionary Scale Modeling) transformer model. Specifically, the facebook/esm2_t6_8M_UR50D model was selected to generate context-aware embeddings.

The feature extraction process transforms variable-length sequences into fixed-length 320-dimensional vectors using mean pooling across residue-level hidden states. This approach captures the biochemical and evolutionary context of the sequence while ensuring a uniform input space for the downstream classifier. The module supports GPU acceleration and handles batch processing through automated padding and truncation.

3. Model Architecture and Information-Accretion Loss

The core predictive engine, developed by Worker-Biol-3, is a Multi-Layer Perceptron (MLP) specifically optimized for high-dimensional sequence embeddings.

  • BPClassifier Architecture: The network incorporates BatchNorm1d and Dropout layers to mitigate overfitting, which is a common risk given the high dimensionality of ESM-2 embeddings relative to specific GO term frequencies.
  • Information-Accretion Loss: Standard Binary Cross-Entropy (BCE) is replaced by a custom InformationAccretionLoss. This loss function scales the penalty for misclassifications based on the individual Information Accretion (IA) weights of the BP nodes. This forces the model to prioritize accuracy on highly informative biological terms.
  • Threshold Optimization: Because the model outputs probabilities for multi-label classification, the pipeline includes logic to iterate through dynamic decision thresholds. This allows for the identification of the optimal threshold that maximizes the weighted F1-measure (wFmax) and minimizes Semantic Distance (Smin).

4. Multi-Tiered Evaluation Framework

To ensure the model is robust across different levels of biological understanding, Worker-Biol-4 implemented a validation framework that categorizes protein targets based on existing knowledge levels:

  1. No-Knowledge: Targets with 0 existing annotations.
  2. Limited-Knowledge: Targets with 1-3 existing annotations.
  3. Partial-Knowledge: Targets with more than 3 existing annotations.

The framework provides automated dataset partitioning and generates performance reports for each category. This segmentation is critical for assessing how well the model generalizes from well-studied proteins to "dark matter" proteins (those with no known functions), mirroring the challenges found in the CAFA (Critical Assessment of Functional Annotation) competitions.

5. Implementation Status

All modules are currently functional and saved within the workers/ directory:

  • compute_ic.py: Handles semantic weights.
  • sequence_feature_engineering.py: Handles ESM-2 embedding generation.
  • bp_term_prediction.py: Contains the MLP architecture and weighted loss function.
  • evaluation_framework.py: Handles tiered performance reporting.

No worker results were marked with low confidence; however, it is noted that the current feature extraction uses the 8M parameter version of ESM-2. For higher accuracy at the cost of computational overhead, larger ESM-2 variants (e.g., 650M or 3B parameters) could be dropped into the existing sequence_feature_engineering.py module with minimal configuration changes.

Note: The complete synthesized pipeline is ready for integration, with all components communicating via standardized JSON and CSV formats.
