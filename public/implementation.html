2. Automated Dispatch Logic & Middleware Code

The following production-ready TypeScript microservice handles incoming webhooks, verifies security signatures, maps data structures, finds available technicians, and schedules jobs in the FSM:

import express, { Request, Response } from 'express';
import crypto from 'crypto';
import axios from 'axios';
const app = express();
app.use(express.json({
verify: (req: any, res, buf) => { req.rawBody = buf; }
}));
const CRM_WEBHOOK_SECRET = process.env.CRM_WEBHOOK_SECRET || 'koehne_secure_hmac_key';
const FSM_API_URL = process.env.FSM_API_URL || 'https://api.fieldservicefsm.com/v1';
const FSM_API_KEY = process.env.FSM_API_KEY || 'fsm_key_live_abc123';
const GHL_API_URL = process.env.GHL_API_URL || 'https://services.gohighlevel.com/v1';
const GHL_API_KEY = process.env.GHL_API_KEY || 'ghl_key_live_xyz789';
function verifyHmacSignature(req: Request): boolean {
const signature = req.headers['x-koehne-signature'] as string;
if (!signature) return false;
const hmac = crypto.createHmac('sha256', CRM_WEBHOOK_SECRET);
// @ts-ignore
const digest = hmac.update(req.rawBody).digest('hex');
return crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(digest));
}
app.post('/api/v1/adee-dispatch', async (req: Request, res: Response) => {
if (!verifyHmacSignature(req)) {
return res.status(401).json({ error: 'Signature verification failed.' });
}
const payload = req.body;
// Make sure the lead is ready to be dispatched
if (payload.event !== 'opportunity.updated' || payload.data.stage !== 'Ready to Dispatch') {
return res.status(200).json({ status: 'ignored', reason: 'Opportunity is not in Dispatch stage.' });
}
try {
const data = payload.data;
// Resolve technician skills based on the requested service type
const skillMapping: Record<string, string[]> = {
'HVAC_REPLACEMENT': ['AC_Install_L2', 'System_Ducting'],
'HVAC_REPAIR': ['AC_Diagnostics', 'EPA_Universal'],
'HEATING_REPAIR': ['Furnace_Diagnostics', 'Gas_Safety_L1']
};
const requestedSkills = skillMapping[data.service_type] || ['General_Service'];
// Map GHL data to your FSM system format
const fsmJobPayload = {
external_ref_id: data.opportunity_id,
job_type: data.service_type === 'HVAC_REPLACEMENT' ? 'Replacement' : 'Service Call',
summary: data.opportunity_name,
description: `Special Instructions: ${data.job_details.special_instructions}`,
duration_minutes: Math.round(data.job_details.estimated_hours * 60),
scheduled_start: `${data.preferred_schedule_date}T08:00:00Z`,
customer_details: {
display_name: `${data.customer.first_name} ${data.customer.last_name}`,
primary_phone: data.customer.phone,
primary_email: data.customer.email,
service_address: {
street_1: data.customer.billing_address.street,
city: data.customer.billing_address.city,
state_province: data.customer.billing_address.state,
postal_code: data.customer.billing_address.postal_code,
country: 'USA'
}
},
skills_required: requestedSkills
};
// Create the job record in your FSM platform
const fsmResponse = await axios.post(`${FSM_API_URL}/jobs`, fsmJobPayload, {
headers: { 'Authorization': `Bearer ${FSM_API_KEY}`, 'Content-Type': 'application/json' }
});
const fsmJob = fsmResponse.data;
// Sync the assigned technician and job details back to GHL
await axios.patch(`${GHL_API_URL}/opportunities/${data.opportunity_id}`, {
field_service_job_id: fsmJob.id,
dispatch_status: 'Scheduled',
assigned_technician_name: fsmJob.technician_name || 'Unassigned',
scheduled_time: fsmJob.scheduled_start
}, {
headers: { 'Authorization': `Bearer ${GHL_API_KEY}`, 'Content-Type': 'application/json' }
});
return res.status(200).json({ status: 'success', job_id: fsmJob.id, assigned_tech: fsmJob.technician_name });
} catch (error: any) {
console.error('[ERROR] Dispatch pipeline failed: ', error.message);
return res.status(500).json({ error: 'System processing failed.', details: error.message });
}
});
app.listen(3000, () => console.log('ADEE Dispatch Service online on Port 3000'));

Step-by-Step Exception Workflows

  • Handling Inventory Shortages: If the automated engine flags an inventory shortage (e.g., "No field technician nearby has a 3-ton evaporator coil in truck stock"), the dispatcher selects "Resolve Match". They then choose the closest technician and select "Assign with Warehouse Pickup detour". The system automatically schedules the warehouse stop and recalculates the customer's arrival time.
  • Emergency Scheduling Overrides: When an emergency call comes in, select "Emergency Priority". The engine will search for the nearest technician and display a scheduling warning (e.g., "Assigning this job will delay Job #382 by 45 minutes."). Click "Confirm Emergency Override" to trigger automated schedule updates and text notifications to both customers.

Edge Case Management & System Failovers

To maintain operational consistency during unexpected scenarios, configure these automated system failovers:

01

1. FSM API Integration Failures (E.g., System Timeout)

How it is detected: The middleware (Make/Zapier) receives an HTTP status code error (500/503) or experiences a connection timeout while attempting to sync a job.

System Action: Implement a standard exponential backoff retry policy (attempting reconnections at 15 seconds, 2 minutes, and 10 minutes). If the error persists, the system sends an automated SMS alert to the dispatcher: "ALERT: Automatic dispatch sync failed for Job #{{opportunity.id}}. Manual scheduling required."

02

2. Invalid Service Addresses

How it is detected: The geocoding API (Google Maps) cannot find coordinates or returns ZERO_RESULTS for a customer-provided address.

System Action: GHL tags the contact as Invalid_Address and sends an automated text to the customer: "Hi {{contact.first_name}}, we had trouble validating your address. Please reply to this text with your exact physical address to confirm your service appointment."

03

3. Real-Time Technician Outages

How it is detected: A technician updates their mobile app status to "Off-Duty" or "Inactive" during an active work shift.

System Action: The system automatically flags their remaining scheduled appointments for the day, returns those jobs to the unassigned queue, and triggers a dashboard notification to the dispatcher.
